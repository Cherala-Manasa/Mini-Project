Java Intermediate Quiz

Test your intermediate knowledge of Java programming.

1. Which of these statements is incorrect about constructors?

A. Constructors can be overloaded
B. Constructors cannot be inherited
C. Constructors have a return type
D. Constructors are called when an object is created
Incorrect. Constructors in Java do not have a return type. They are special methods called when an object is created.
Correct! Constructors do not have a return type, and they are used to initialize objects.

2. What is the default value of a local variable in Java?

A. 0
B. null
C. Garbage value
D. Local variables do not have a default value
Incorrect. Local variables in Java do not have a default value and must be explicitly initialized before use.
Correct! Local variables in Java do not have a default value and need to be explicitly initialized.

3. What will be the output of the following code?


            public class Test {
                public static void main(String[] args) {
                    int x = 10;
                    int y = 20;
                    System.out.println(x + y + " " + (x - y) + " " + x * y);
                }
            }
A. 30 10 200
B. 30 -10 200
C. 30 -10 200.0
D. Compilation error
Incorrect. The code will output "30 -10 200". The expression `x + y` results in "30", and `(x - y)` results in "-10", with `x * y` producing "200".
Correct! The output is "30 -10 200" as the `+` operator concatenates the numbers and strings in the output.

4. Which of the following statements is true regarding the `this` keyword in Java?

A. `this` can be used to invoke the current class method
B. `this` can be used to invoke the current class constructor
C. `this` can be passed as an argument in the method call
D. All of the above
Incorrect. `this` keyword in Java can be used for all the mentioned uses: invoking methods, constructors, and passing as an argument.
Correct! The `this` keyword can be used to invoke methods and constructors within the same class, and it can also be passed as an argument.

5. Which of the following is not a feature of the Java programming language?

A. Object-oriented
B. Platform-independent
C. Single inheritance
D. Garbage collection
Incorrect. Java supports multiple inheritance through interfaces and has garbage collection, among other features.
Correct! Single inheritance is not a feature of Java; it supports multiple inheritance through interfaces.

6. What will be the output of the following code?


            public class Test {
                public static void main(String[] args) {
                    int x = 5;
                    int y = 10;
                    System.out.println(++x * y--);
                    System.out.println(x);
                    System.out.println(y);
                }
            }
A. 60, 5, 10
B. 50, 6, 9
C. 60, 6, 9
D. 50, 5, 10
Incorrect. The output is 60, 6, 9 because `++x` increments `x` before multiplication, and `y--` decrements `y` after multiplication.
Correct! The code prints 60 (result of `++x * y--`), then `x` is 6 and `y` is 9 after the operations.

7. What will be the output of the following code?


            public class Test {
                public static void main(String[] args) {
                    System.out.println("Hello" + 1 + 2 + 3);
                    System.out.println(1 + 2 + 3 + "Hello");
                }
            }
A. Hello123, 6Hello
B. Hello6, 6Hello
C. Hello123, Hello123
D. Hello6, Hello6
Incorrect. The first line concatenates "Hello" with the numbers resulting in "Hello123", while the second line performs arithmetic before concatenating "Hello".
Correct! The first output is "Hello123" and the second is "6Hello".

8. The StringBuilder class is immutable.

A. True
B. False
Incorrect. The `StringBuilder` class is mutable, meaning its content can be changed after it is created.
Correct! The `StringBuilder` class is mutable, unlike `String`, which is immutable.

9. What will be the output of the following code?


            public class Test {
                public static void main(String[] args) {
                    String str1 = "Hello";
                    String str2 = "Hello";
                    String str3 = new String("Hello");
                    System.out.println(str1 == str2);
                    System.out.println(str1 == str3);
                    System.out.println(str1.equals(str3));
                }
            }
A. true, false, true
B. false, false, true
C. true, true, false
D. false, true, true
Incorrect. The `==` operator compares references, so `str1` and `str2` refer to the same object, while `str3` is a new object. The `equals` method compares content.
Correct! `str1 == str2` is true (same reference), `str1 == str3` is false (different reference), and `str1.equals(str3)` is true (same content).

10. Identify the error in the following code


            public class Test {
                public static void main(String[] args) {
                    int x = 5;
                    int y = 0;
                    System.out.println(x / y);
                }
            }
A. ArithmeticException: / by zero
B. ArrayIndexOutOfBoundsException
Incorrect. Division by zero in Java will result in an `ArithmeticException`.
Correct! The code will throw an `ArithmeticException` because division by zero is not allowed in Java.
