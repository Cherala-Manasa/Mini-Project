Advanced JavaScript Quiz

Test your advanced knowledge of JavaScript.

1. What will be the output of the following code?

console.log([] + []);
A. []
B. [object Object]
C. ''
D. Error
Incorrect. The correct answer is ''. When you add two empty arrays together, JavaScript converts them to strings and concatenates them, resulting in an empty string.
Correct! When you add two empty arrays together, JavaScript converts them to strings and concatenates them, resulting in an empty string.

2. What is the result of the following code snippet?

console.log(1 == '1');
A. true
B. false
C. TypeError
D. undefined
Incorrect. The correct answer is true. The == operator performs type coercion and treats the string '1' as the number 1.
Correct! The == operator performs type coercion and treats the string '1' as the number 1, resulting in true.

3. What will this code output?

console.log(typeof NaN);
A. 'number'
B. 'NaN'
C. 'undefined'
D. 'object'
Incorrect. The correct answer is 'number'. NaN stands for 'Not-a-Number', but it is still considered a number type in JavaScript.
Correct! NaN stands for 'Not-a-Number', but it is still considered a number type in JavaScript.

4. What does the following code do?

console.log([...new Set('hello')].join(''));
A. hello
B. helo
C. hll
D. Error
Incorrect. The correct answer is 'helo'. The Set object removes duplicate letters, so 'hello' becomes 'helo'.
Correct! The Set object removes duplicate letters, so 'hello' becomes 'helo'.

5. How does JavaScript handle asynchronous code?

A. Synchronously using callbacks
B. Using Promises and async/await
C. Using only setTimeout
D. None of the above
Incorrect. The correct answer is using Promises and async/await. These provide a more readable and efficient way to handle asynchronous operations in JavaScript.
Correct! Using Promises and async/await provides a more readable and efficient way to handle asynchronous operations in JavaScript.

6. What is the result of this code?

function* generator() {
  yield* [1, 2];
  yield 3;
}
const gen = generator();
console.log([...gen]);
A. [1, 2, 3]
B. [1, 2]
C. [2, 3]
D. [1, 2, 1, 2, 3]
Incorrect. The correct answer is [1, 2, 3]. The generator function yields values from the array [1, 2] and then yields 3.
Correct! The generator function yields values from the array [1, 2] and then yields 3, resulting in [1, 2, 3].

7. What will the following code output?

console.log(typeof {});
A. 'object'
B. 'undefined'
C. 'function'
D. 'object Object'
Incorrect. The correct answer is 'object'. In JavaScript, the typeof operator returns 'object' for objects.
Correct! In JavaScript, the typeof operator returns 'object' for objects.

8. What is the output of the following code?

let arr = [1, 2, 3];
arr.length = 0;
console.log(arr);
A. []
B. [1, 2, 3]
C. [undefined, undefined, undefined]
D. Error
Incorrect. The correct answer is []. Setting the length property of an array to 0 removes all elements from the array.
Correct! Setting the length property of an array to 0 removes all elements from the array, resulting in an empty array.

9. What will the following code print?

const obj = { a: 1, b: 2 };
console.log(Object.keys(obj));
A. ['a', 'b']
B. ['b', 'a']
C. ['a', 'b', '1', '2']
D. Error
Incorrect. The correct answer is ['a', 'b']. The Object.keys() method returns an array of a given object's own enumerable property names.
Correct! The Object.keys() method returns an array of a given object's own enumerable property names, in the same order as that provided by a for...in loop.

10. What will this code output?

const x = { a: 1 };
const y = { ...x };
y.a = 2;
console.log(x.a);
A. 1
B. 2
C. undefined
D. Error
Incorrect. The correct answer is 1. The spread operator creates a shallow copy of the object, so changes to the copy do not affect the original.
Correct! The spread operator creates a shallow copy of the object, so changes to the copy do not affect the original. Therefore, x.a remains 1.
